# Review 模块技术方案

> 龙头回测验证系统 — 验证四维评分的实际表现

---

## 一、背景

`dragon-quant` 每天 scan 产出 topN 龙头的四维评分，写入 `dragons` 表。但这些龙头在后续交易日实际表现如何？评分高的票是否真的有更大机会？需要一套回测机制来量化验证。

## 二、核心逻辑

```
scan 执行时                       review 执行时（用户手动触发）
───────                          ──────────
写入 dragons                     读取 pending 记录
(trade_date + 基本面快照)          ↓
      ↓                          拉取历史日K（雪球）
5 日窗口去重                      找入选日后第一个非涨停日 → 买入价
      ↓                          计算买入后 5 日最大收益/回撤
buy_date = NULL                   ↓
review_status = 'pending'         UPDATE dragons
```

### 2.1 入选去重

同一标的在 **5 个交易日内** 只写入 `dragons` 一次。

- 金螳螂 5 月 6 日首次入选 → `trade_date = 2026-05-06`
- 5 月 7 日再次入选 → 距上次 1 个交易日，**跳过**
- 5 月 12 日再次入选 → 距上次 4 个交易日，**跳过**
- 5 月 13 日再次入选 → 距上次 5 个交易日，**允许写入**

实现：`orchestrator.py` 写入前调 `db.get_last_entry(code)` 获取上次 `trade_date`，用交易日历计算间隔 (<5 则跳过)。

### 2.2 交易日历

基于雪球平安银行（000001）日K构建：

```python
calendar = build_trade_calendar("2026-05-01", "2026-05-31")
# → {"2026-05-06", "2026-05-07", "2026-05-08", ...}  # 只含交易日

trade_days_between("2026-05-06", "2026-05-13", calendar)
# → 5  (05-07, 05-08, 05-11, 05-12, 05-13)
```

优点：不依赖任何外部假期表，日K天然过滤了周末和节假日。

### 2.3 买入时机

从入选日 `trade_date` **之后** 开始逐日查找，找到第一个非涨停日作为买入日。

涨停判定规则：

```python
def is_limit_up(kline):
    return kline.pct >= 9.9 and kline.close == kline.high
```

- 涨幅 ≥ 9.9% 且封板到收盘 → 涨停，买不进，跳过
- 否则 → 可买入，`buy_price = 该日日K最低价`

5 日内找不到断板日 → `review_status = 'no_entry'`（无买入机会）。

### 2.4 5 日回测

买入日后取 5 个交易日日K：

```
max_return_5d  = (买入后 5 日最高价 - buy_price) / buy_price × 100%
max_drawdown_5d = (买入后 5 日最低价 - buy_price) / buy_price × 100%
```

## 三、DB Schema

`dragons` 表新增 5 列：

| 列名 | 类型 | 说明 |
|------|------|------|
| `buy_date` | TEXT | 买入日 YYYY-MM-DD |
| `buy_price` | REAL | 买入价（断板日日K最低价） |
| `max_return_5d` | REAL | 5 日最大收益率 (%) |
| `max_drawdown_5d` | REAL | 5 日最大回撤率 (%) |
| `review_status` | TEXT | pending / completed / no_entry / error |

新增索引：`idx_dragons_review ON dragons(review_status, trade_date)`

迁移策略：幂等 `ALTER TABLE ADD COLUMN`（已有 `_ensure_schema` 中的迁移模式）。

## 四、CLI 命令

```bash
# 回测全部待处理记录
dragon-quant review

# 指定日期
dragon-quant review --date 20260519

# 只回测 top 5
dragon-quant review --top 5

# 强制重算（无视 review_status）
dragon-quant review --force --date 20260519
```

```
dragon_quant/

```

## 六、边界情况

| 场景 | 处理 |
|------|------|
| 入选后连续 5 日涨停 | `review_status = 'no_entry'` — 无买入机会 |
| review 时 K 线不足 5 日 | 只算已有数据，`status = 'completed'`（部分数据） |
| `--force` 强制重算 | 自 `review_status='pending'` 中筛选（暂不支持全量） |
| 002 / 北交所 30cm 涨停 | 入选前已过滤（`_is_valid_candidate`），不进入 dragons |
| 停牌期间 | 日K无数据，`find_entry_day` 自动跳过无K日期 |
| 买入当天再涨停 | `find_entry_day` 要求 `close != high`，当天有交易机会即可买入 |

## 七、评审结论

- **回测操作是异步的**：系统不自动勾采数据，自己 run review
- **评分和回测结果独立存放**：各维度评分在 `scan_stocks` 中，回测结果在 `dragons` 中
- **盈亏只看 5 个交易日**：不计算长期持仓，只验证 5 日内的表现

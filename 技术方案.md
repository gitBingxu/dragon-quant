## 一、API 接口文档

### 1.1 东财 — 概念板块涨跌幅排行

**用途：** 获取当日所有概念板块的涨跌幅排行，筛选前10涨/前10跌板块

**接口：** `GET push2.eastmoney.com/api/qt/clist/get`

**格式：** JSONP（需带 `cb` 回调参数）

**请求参数：**

| 参数 | 值 | 说明 |
|------|-----|------|
| `fs` | `m:90+t:3` | 概念板块市场 |
| `fltt` | `1` | 数据类型 |
| `invt` | `2` | 投资类别 |
| `fid` | `f3` | 排序字段（涨跌幅） |
| `po` | `1`（涨幅榜） / `0`（跌幅榜） | 正序/倒序 |
| `pn` | `1` | 页码 |
| `pz` | `500` | 每页条数 |
| `fields` | `f12,f14,f3,f4,f8` | 返回字段 |
| `ut` | `fa5fd1943c7b386f172d6893dbfba10b` | 用户token |
| `cb` | `jQuery_xxx` | JSONP 回调名 |

**返回字段：**

| 字段 | 含义 | 示例 |
|------|------|------|
| `f12` | 板块代码 | `BK1145` |
| `f14` | 板块名称 | `机器人执行器` |
| `f3` | 涨跌幅% | `5.31` |
| `f4` | 涨跌额 | `12.50` |
| `f8` | 振幅% | `5.63` |
| `f104` | 换手率% | `35.00` |

**curl 示例：**
```bash
# 涨幅榜（po=1）
curl 'https://push2.eastmoney.com/api/qt/clist/get?np=1&fltt=1&invt=2&cb=jQuery_test&fs=m%3A90%2Bt%3A3&fields=f12%2Cf14%2Cf3%2Cf4%2Cf8&fid=f3&pn=1&pz=500&po=1&ut=fa5fd1943c7b386f172d6893dbfba10b&dect=1&wbp2u=%7C0%7C0%7C0%7Cweb' \
  -H 'Referer: https://quote.eastmoney.com/center/hsbk.html' \
  -b 'qgqp_b_id=xxx; st_nvi=xxx' \
  -H 'User-Agent: Mozilla/5.0'
```

### 1.2 东财 — 概念板块成分股

**用途：** 获取指定概念板块（如 BK1145）内所有个股的实时行情，按涨跌幅排序。领涨板块的前N名为候选股。

**接口：** `GET push2.eastmoney.com/api/qt/clist/get`

**格式：** JSONP

**请求参数：**

| 参数 | 值 | 说明 |
|------|-----|------|
| `fs` | `b:BK1145+f:!` | `b:BKxxx` 指定板块，`f:!` 返回全部股票 |
| `fid` | `f3` | 按涨跌幅排序 |
| `po` | `1` | 降序（涨跌幅从高到低） |
| `pn` | `1` | 页码（分页） |
| `pz` | `50` | 每页条数（板块成分股通常不超过50） |
| `fields` | `f12,f14,f3,f2,f4,f8` | 返回字段 |
| `fltt` | `1` | |
| `invt` | `2` | |
| `ut` | `fa5fd1943c7b386f172d6893dbfba10b` | |
| `cb` | `jQuery_xxx` | JSONP 回调 |

> **注意：** `f:!` 中的 `!` 在 URL 中编码为 `\u0021` 或 `%21`。不同平台需正确处理编码。

**返回字段：**

| 字段 | 含义 | 示例 |
|------|------|------|
| `f12` | 股票代码 | `300056` |
| `f14` | 股票名称 | `中创环保` |
| `f2` | 最新价 | `15.60` |

| `f4` | 涨跌额 | `0.59` |
| `f8` | 振幅% | `5.20` |

### 1.3 东财 — 概念板块5分钟K线

**用途：** 获取指定概念板块（如 BK1145）的5分钟K线数据，用于资金承接跨板块虹吸检测。

**接口：** `GET push2his.eastmoney.com/api/qt/stock/kline/get`

**格式：** JSONP

**请求参数：**

| 参数 | 值 | 说明 |
|------|-----|------|
| `secid` | `90.BK1145` | `90` = 概念板块市场，`BKxxx` = 板块代码 |
| `klt` | `5` | K线类型：5=5分钟 |
| `fqt` | `1` | 复权类型 |
| `beg` | `0` | 起始时间戳 |
| `end` | `20500101` | 结束日期 |
| `lmt` | `1000000` | 返回条数上限 |
| `fields1` | `f1,f2,f3,f4,f5,f6` | |
| `fields2` | `f51,f52,f53,f54,f55,f56,f57,f58,f59,f60,f61` | K线字段 |
| `ut` | `fa5fd1943c7b386f172d6893dbfba10b` | |
| `cb` | `jQuery_xxx` | JSONP 回调 |

**返回格式：**
```json
{
  "data": {
    "klines": [
      "2026-05-08 09:35,1400.00,1402.50,1405.00,1398.50,123456,9876543.21,1.20,0.50,6.50,2.30",
      ...
    ]
  }
}
```

**K线字段顺序：** `时间,开,收,高,低,成交量,成交额,振幅,涨跌幅,涨跌额,换手率`

**curl 示例：**
```bash
curl 'https://push2his.eastmoney.com/api/qt/stock/kline/get?cb=jQuery_test&secid=90.BK1145&ut=fa5fd1943c7b386f172d6893dbfba10b&fields1=f1%2Cf2%2Cf3%2Cf4%2Cf5%2Cf6&fields2=f51%2Cf52%2Cf53%2Cf54%2Cf55%2Cf56%2Cf57%2Cf58%2Cf59%2Cf60%2Cf61&klt=5&fqt=1&beg=0&end=20500101&lmt=100' \
  -H 'Referer: https://quote.eastmoney.com/bk/90.BK1145.html' \
  -b 'qgqp_b_id=xxx; st_nvi=xxx' \
  -H 'User-Agent: Mozilla/5.0'

### 1.4 雪球 — 个股日K线

**用途：** 获取个股日K线数据，用于判断连板数、跳水日、反弹弹性。

**接口：** `GET stock.xueqiu.com/v5/stock/chart/kline.json`

**请求参数：**

| 参数 | 值 | 说明 |
|------|-----|------|
| `symbol` | `SH600519` | 股票代码，前缀 SH/SZ |
| `period` | `day` | K线周期 |
| `type` | `before` | 向前取数据 |
| `count` | `-20` | 取20根（负数表示从最新往前） |
| `indicator` | `kline` | 指标类型，必填 |
| `begin` | 时间戳（毫秒） | 起始时间 |

**请求头：** 需携带完整 Cookie（`xq_a_token`, `xq_r_token`, `xq_is_login`, `u` 等）

**返回格式：**
```json
{
  "data": {
    "item": [
      [timestamp, volume, open, high, low, close, chg, pct, turnover, amount],
      ...
    ]
  }
}
```

**字段顺序：** `[时间戳ms, 成交量, 开盘, 最高, 最低, 收盘, 涨跌额, 涨跌幅%, 换手率, 成交额]`

**curl 示例：**
```bash
curl 'https://stock.xueqiu.com/v5/stock/chart/kline.json?symbol=SH600519&period=day&type=before&count=-20&indicator=kline&begin=1775577600000' \
  -H 'User-Agent: Mozilla/5.0 (Linux; Android 6.0; Nexus 5 Build/MRA58N) AppleWebKit/537.36' \
  -H 'Referer: https://xueqiu.com/' \

### 1.7 腾讯 — 个股1分钟分时（5分K fallback）

**用途：** 当雪球5分钟K线失效时，从腾讯拉取1分钟分时数据，合成为5分钟K线（5根合成1根）。

**接口：** `GET web.ifzq.gtimg.cn/appstock/app/minute/query?code=sh600519`

**返回格式：** JSON

| 字段 | 含义 |
|------|------|
| `data.sh600519.qt` | 股票行情快照 |
| `data.sh600519.data.data` | 1分钟分时数组 |

**分时数据格式：** `"0930 1371.66 186 25512876.20"` → `时间 价格 成交量 成交额`

**合成算法：** 每5根1分钟bar合成1根5分钟K线（取第1根的开盘、最后根的收盘、期间最高/最低、成交量/额求和）

**curl 示例：**
```bash
curl -s 'https://web.ifzq.gtimg.cn/appstock/app/minute/query?code=sh600519'

## 二、请求发起顺序

### Step-by-Step 调用序列

```
第1步 [串行] — 东财·板块排行（2次）
  ├─ 东财 涨幅榜 po=1, pz=500      → 前10涨板块
  └─ 东财 跌幅榜 po=0, pz=500      → 前10跌板块

第2步 [串行·RateLimited] — 东财·板块成分股（20次）
  ├─ 遍历前10涨板块 → 东财·成分股（po=1 涨跌幅降序，取前5只）
  │   ├─ 过滤：非ST、非30/68开头
  │   └─ 记录：每只候选股出现在哪些领涨板块中
  └─ 遍历前10跌板块 → 东财·成分股（用于资金承接检测）

第3步 [串行·RateLimited] — 雪球·日K线（N+1次）
  ├─ 雪球·候选股日K线（N次，N=去重候选股数）
  │   └─ period=day, count=-30，计算每只连板高度
  └─ 雪球·大盘指数日K线（1次）
      └─ 上证指数 symbol=SH000001，用于抗跌性跳水日判断

第4步 [串行] — 候选排序
  └─ 按(概念数量降序, 连板高度降序) → 取前25

第5步 [并发·RateLimiter] — 评分数据加载
  ├─ T1: 东财·20个板块5分钟K线（串行20次）
  │     secid=90.BKxxx, klt=5, lmt=100
  ├─ T2: 雪球·个股5分钟K线（串行25次）
  │     period=5m, count=-96，用于封板时间/日内承接
  └─ T3: 腾讯·实时行情（串行batch）
        qt.gtimg.cn/q=shxxx 批量查询，用于评分校验
        qt.gtimg.cn/q=shxxx 批量查询，用于评分校验

第6步 [串行] — 聚合
  └─ 全部数据写入共享 JSON 文件
     └─ 写入到 get_data_dir() / shared / dq_shared_YYYYMMDD.json

第7步 [subprocess] — 并行打分
  ├─ 子进程1: 候选股A → 四维并行评分 → stdout
  ├─ 子进程2: 候选股B → 四维并行评分 → stdout
  ├─ ...（workers 个并发）
  └─ 所有子进程：只读共享缓存，零网络调用

第8步 [串行] — 输出
  └─ 主进程收集结果 → 加权排序 → 输出报告
```

**统计：**

| 步骤 | 调用次数 | 耗时瓶颈 |
|------|---------|---------|
| 第1步 板块排行 | 2 | 无，极快 |
| 第2步 板块成分股 | 20 | ⚠️ 东财串行，约20×0.5s=10s |
| 第3步 候选池日K | ~50 | ⚠️ 雪球串行，约50×0.3s=15s |
| 第5步 评分数据加载 | ~50 | ✅ 4线程并发，约15s |
| 第7步 subprocess | workers 个 | ✅ 并行，约5-10s |
| **总计** | **~120** | **约40-50秒** |

**候选股筛选规则详解：**

| 步骤 | 说明 |
|------|------|
| ① 取前10涨板块 | 东财板块涨幅榜，po=1，按涨跌幅降序 |
| ② 每板块取成分股 | 东财·成分股，po=1 涨跌幅降序，取前5只 |
| ③ 过滤 | 剔除ST/*ST、30xxxx（创业板）、68xxxx（科创板） |
| ④ 多板块跟踪 | 例：金螳螂出现在半导体和商业航天两个板块中 → 记录 `金螳螂: [半导体, 商业航天]` |
| ⑤ 连板高度 | 雪球日K线 → 判断连续涨停天数 |
| ⑥ 排序 | 先按概念数量降序，再按连板高度降序，取前25 |

```python
# 候选股排序伪代码
class Candidate:
    code: str
    concepts: list[str]       # 所属领涨概念列表
    board_count: int          # 连板高度

def rank(candidates: list[Candidate]) -> list[Candidate]:
    # 概念数量权重 > 连板高度权重
    return sorted(candidates,
        key=lambda c: (len(c.concepts), c.board_count),
        reverse=True)[:25]
```

## 三、反爬/限流分析

### 3.1 限流关键 Header

基于东财和雪球的实际请求对比，以下 Header 用于反爬/限流检测：

| Header | 风险等级 | 原因 |
|--------|---------|------|
| **`Cookie`** | 🔴 核心 | 没 Cookie 直接空响应 |
| **`User-Agent`** | 🔴 核心 | 经典 bot 检测指标，缺了就 403 |
| **`sec-ch-ua`** | 🔴 核心 | 现代浏览器自动带的浏览器指纹，Python/curl 默认没有 |
| **`sec-ch-ua-platform`** | 🔴 核心 | 与 sec-ch-ua 成对出现 |
| **`Sec-Fetch-Dest`** | 🔴 核心 | Fetch Metadata 规范，浏览器自动发，爬虫不发 |
| **`Sec-Fetch-Mode`** | 🔴 核心 | 同上 |
| **`Sec-Fetch-Site`** | 🔴 核心 | 同上 |
| **`Referer`** | 🟡 重要 | 各接口有固定值，不匹配可能被拒 |
| **`Origin`** | 🟡 重要 | 雪球要求 `https://xueqiu.com` |
| `Accept-Language` | 🟢 辅助 | 可能参与指纹计算，但不是决定性 |
| `sec-ch-ua-mobile` | 🟢 辅助 | 固定值 `?0`，配合 sec-ch-ua 系列 |
| `Accept` / `Cache-Control` / `Connection` / `Pragma` | ⚪ 无关 | 标准头，默认值即可 |

**8 个必须每请求都带的 Header：**

```
Cookie
User-Agent
sec-ch-ua
sec-ch-ua-platform
Sec-Fetch-Dest
Sec-Fetch-Mode
Sec-Fetch-Site
Referer
```

> ⚠️ 不能使用 `urllib.request` 的默认请求头，必须逐个手动设置。

### 两种获取方式

#### 方式一：手动（用户提供给 Agent）

用户通过浏览器 F12 → Network 复制完整 Cookie 后交给 Agent，Agent 写入对应文件。

#### 方式二：无头浏览器自动获取

使用 Playwright 打开东财/雪球页面，监听 HTTP 请求，从请求头中提取 Cookie。

```python
# 伪代码示例
async def fetch_cookies(url: str, domain: str) -> str:
    browser = await playwright.chromium.launch()
    page = await browser.new_page()
    cookies = []
    
    page.on("request", lambda req: cookies.append(req.headers.get("cookie", "")))
    await page.goto(url)
    await page.wait_for_timeout(5000)  # 等待页面加载和请求发出
    
    await browser.close()
    return max(cookies, key=len)  # 取最完整的 Cookie
```

使用场景：
- **东财：** 直接打开 `https://quote.eastmoney.com/center/hsbk.html`，无需登录
- **雪球：** 需要已登录的浏览器上下文（Playwright 可以复用用户浏览器数据）

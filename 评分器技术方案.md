## 二、带动性 scorer（权重 35%）— `scorers/drive.py`

### 2.1 核心问题
这只股票封板时，板块里其他股票跟不跟？

### 2.2 输入数据

| 缓存 key | 内容 | 用途 |
|----------|------|------|
| `kline:day:{code}` | 个股日K线（近 30 天） | 找近 3 个涨停日、连板数、换手率 |
| `kline:5min:{code}` | 个股 5 分 K（当天） | 封板时间 |
| `sector:components:{s_code}` | 板块全部成分股 | 板块总规模、按 code 查成分股 |
| `quotes:batch` | 全部股票实时行情 | 成分股当日涨跌幅 |

**数据策略确认**：不拉全部成分股历史日K。从 Phase B 已获取的 10 个领涨板块 × 5 只 候选股（50 只去重）的 5 分 K 中比对封板时间，在 50 只候选股范围内做板块内排序。板块总成分股数量从 `sector:components` 获取，涨停/跟风判断从 `quotes:batch` 获取。

### 2.3 算法步骤

#### Step 1 — 找近 3 个涨停日

从日K线倒推，找最近 3 个涨停日，构造 `limit_up_dates`：

```python
limit_up_dates = []  # [{date, board_time, turnover, consecutive}, ...]
```

- 涨停判定：`pct ≥ 9.9%`
- 从 `kline:5min:{code}` 找封板时间：遍历 bar，找到首根 `close ≥ prev_close × 1.099` 的 bar 的时间戳
- 如果没有涨停日 → 返回 **30 分兜底**（非 0 分，避免拉低综合过狠）

#### Step 2 — 对每个涨停日，独立打分

**每日带动性 = A × 0.30 + B × 0.30 + C × 0.40**

##### (a) 板块共鸣度 Voice（30%）

```
共鸣率 = 同板块涨停股数量 / 板块成分股总数
voice_score = min(共鸣率 / 0.10, 1.0) × 100
```

- 同板块涨停股 ≥ 10% → 满分 100
- 例：板块 50 只，5 只涨停 → 5/50=10% → 满分

**涨停判断**：当日从 `quotes:batch` 中查找板块成分股 pct ≥ 9.9% 的数量。历史涨停日用 `kline:day:{code}` 的当日 pct 判断（仅在 50 只候选股范围内）。

##### (b) 板块跟风力度 Follow（30%）

```
跟风率 = 非涨停股中涨幅 > 3% 的数量 / (板块总成分股 - 涨停股数量)
follow_score = min(跟风率 / 0.15, 1.0) × 100
```

- 非涨停且涨 >3% 的占比 ≥ 15% → 满分 100
- 分母排除已涨停的，只看"被带动但没封板"的

**数据来源**：当日用 `quotes:batch`，历史涨停日仅用候选股池内数据估算。

##### (c) 封板决策力 Board Leadership（40%）

四个子因子：

```
board_score = rank_score × 0.25 + early_bonus × 0.25 + gap_score × 0.50
board_score = board_score × word_penalty  (一字板惩罚)
```

| 子因子 | 算法 |
|--------|------|
| **C1 排名分** (25%) | `(1 - (rank-1) / total) × 100`。在同板块涨停候选股中按封板时间排序，排第一 = 100 分。仅在同板块的 5 只候选股范围内排序。 |
| **C2 绝对时间** (25%) | `≤9:30 → 100`, `≤10:30 → 70`, `≤11:30 → 40`, `午后 → 10`。离散阶梯打分。 |
| **C3 小弟紧密度** (50%) | 同板块其他涨停候选股与候选股封板的时间差。平均 ≤5min 且 >50% 在 5min 内 → 100 分；否则 `max(0, 100 - avg_gap/30 × 100)` |
| **C4 一字板惩罚** | `board_time=None` 且换手率 < 1% → 乘 0.85。一字板堵死无带动，打折。 |

**极端情况**：
- 该股在同板块候选股中是唯一的涨停股（无小弟）→ `gap_score = 50`（独苗中性）
- 板块只有 1 只涨停股（自己）→ 板块共鸣 0，C1 封板排名称王（满分），C3=50

##### (d) 连板加分

```python
max_cons = max(date["consecutive"] for date in limit_up_dates)
if max_cons >= 2:
    drive_score = min(drive_score + max_cons * 5, 100)
```

- 每多一连板 +5 分，上限 100
- 2 连板 +10，3 连板 +15…

#### Step 3 — 取近 3 个涨停日中最佳一天的得分

```
最终得分 = max(涨停日1得分, 涨停日2得分, 涨停日3得分)
```

### 2.4 边界情况

| 情况 | 处理 |
|------|------|
| 候选股无涨停日 | 30 分兜底 |
| 板块只 1 只涨停（自己） | 板块共鸣=0，C1=100（称王），C3=50（独苗） |
| 一字板首日 | board_score × 0.85 |
| 成分股数量为 0 | 30 分（数据异常） |

### 2.5 输出

```python
ScoreResult(
    dim="drive",
    score=68.5,
    weight=0.35,
    details={
        "board_echo": 85.0,
        "follow_strength": 60.0,
        "seal_decision": 70.0,
        "seal_rank": 2,              # 同板块涨停候选股中排第2
        "seal_time": "09:45",
        "consecutive_bonus": 10,
        "best_day": "2026-05-08",
        "is_yiziban_penalty": False,
    }
)
```

## 三、抗跌性 scorer（权重 15%）— `scorers/anti_drop.py`

### 3.1 核心问题
大盘跳水（单日跌幅 < -0.7%）时，这只票是跟跌还是硬扛？

### 3.2 输入数据

| 缓存 key | 内容 | 用途 |
|----------|------|------|
| `kline:day:{code}` | 个股日K线（近 30 天） | 个股涨跌幅/高低收 |
| `kline:day:000001` | 上证指数日K线（近 30 天） | 大盘涨跌幅 |
| `kline:5min:{code}` | 个股 5 分 K | 日内走势（下影线/收盘位置） |

### 3.3 算法步骤

#### Step 1 — 识别跳水日

遍历近 30 天上证日K线，找出所有 **大盘涨跌幅 < -0.7%** 的交易日：

> 阈值 -0.7% 是权衡结果：太窄（-1.5%）跳水日太少无法评估；太宽（-0.3%）误判普通震荡。

如果没有跳水日 → 直接返回 **50 分**（中性）。

#### Step 2 — 对每个跳水日做三维评估

**单日得分 = A × 0.40 + B × 0.30 + C × 0.30**

##### (a) 相对回撤强度（40%）

```python
stock_return = 个股当日涨跌幅
excess_return = stock_return - market_pct

if stock_return > 0:
    score = 100              # 逆势收红
elif excess_return > 0:
    score = 60 + excess_return / abs(market_pct) * 40   # 超额为正
elif stock_return > -0.02:
    score = 30               # 小幅跟跌
else:
    score = 0                # 崩了
```

分段判断优于线性公式：个股跌 2% 大盘跌 3% 时，超额收益 +1%，得 60+1/3×40=73 分（合理）。线性 `100+超额×5` 在超额+1% 时只得 100+5=105→封顶（失真）。

##### (b) 日内承接强度（30%）

```python
open_px  = kline.open
close_px = kline.close
high_px  = kline.high
low_px   = kline.low
prev_close = 昨收

# 下影线占比：实体下沿到最低 / 全日振幅
lower_shadow_pct = (min(open_px, close_px) - low_px) / (high_px - low_px)

# 收盘位置：收盘相对于开盘在日内的位置
close_pos = (close_px - open_px) / (high_px - low_px) if high_px != low_px else 0.5

# 日内最大跌幅（相对昨收）
max_drop = (low_px - prev_close) / prev_close
penalty = clamp(abs(max_drop) / 0.05, 0, 1)   # 跌幅 >5% → 惩罚系数=1

support_score = (lower_shadow_pct * 0.6 + close_pos * 0.4) * 100
support_score = support_score * (1 - penalty * 0.3)
support_score = clamp(support_score, 0, 100)
```

**原理**：下影线长 → 盘中有人抄底（60% 权重）。收盘位置高 → 尾盘没崩（40% 权重）。日内最大跌幅 >5% → 盘中确实很惨 → 扣分。

##### (c) 反弹弹性（30%）

看跳水日 **下一个交易日**：

```python
t1_stock_pct  = kline[i+1].pct      # 次日个股涨跌幅
t1_market_pct = 上证kline[i+1].pct   # 次日大盘涨跌幅
alpha = t1_stock_pct - t1_market_pct

if t1_stock_pct > 0 and alpha > 0:
    score = min(alpha / 0.03 * 100, 100)     # 一起涨但跑赢
elif t1_stock_pct > 0 and t1_market_pct <= 0:
    score = 100                               # 独涨最强
elif t1_stock_pct < 0 and t1_market_pct < 0:
    score = max(0, (1 - abs(alpha) / 0.03) * 100)  # 一起跌但相对抗
else:
    score = 0      # 个股跌大盘涨
```

- 次日无数据（跳水日是最近一天）→ 给中性 **50 分**

#### Step 3 — 单日综合

```
单日得分 = 相对回撤 × 0.4 + 日内承接 × 0.3 + 反弹弹性 × 0.3
```

#### Step 4 — 多日汇总

```
最终得分 = 所有跳水日得分的算术平均
```

**连续暴跌加成**：若存在连续 ≥2 个跳水日，且期间 `个股累计跌幅 < 大盘累计跌幅 × 0.5` → **+10 分**（封顶 100）。

### 3.4 边界情况

| 情况 | 处理 |
|------|------|
| 无跳水日 | 50 分 |
| 仅 1 个跳水日 | 忽略连续暴跌加成 |
| 跳水日是最近一天 | 反弹弹性 = 50 分 |

### 3.5 输出

```python
ScoreResult(
    dim="anti_drop",
    score=72.0,
    weight=0.15,
    details={
        "plunge_days": ["2026-04-28", "2026-05-05"],
        "plunge_day_scores": [68.5, 75.5],
        "avg_relative_retreat": 72.0,
        "avg_intraday_hold": 65.0,
        "avg_rebound": 78.0,
        "consecutive_plunge_bonus": 0,
    }
)
```

## 四、领涨性 scorer（权重 25%）— `scorers/leadership.py`

### 4.1 核心问题
不看涨停日，平时这票在同行业里是不是领跑的？个股是否先于板块拉伸？

### 4.2 输入数据

| 缓存 key | 内容 | 用途 |
|----------|------|------|
| `sector:components:{primary_sector}` | 板块全部成分股 | 行业规模、按 code 查成分股 |
| `quotes:batch` | 全部股票实时行情 | 成分股当日涨跌幅 |
| `kline:day:{code}` | 个股日K线（近 30 天） | 历史涨跌幅、排除涨停日 |
| `kline:5min:{code}` | 个股 5 分 K | 个股日内走势（lead-lag 检测） |
| `kline:5min:sector:{s_code}` | 板块 5 分 K | 板块日内走势（lead-lag 检测） |

**数据策略**：历史5日的成分股排名用当日 `quotes:batch` 的 median/std 做正态分布近似（方案A）。

### 4.3 算法步骤

#### Part 1 — 当日真实排名

```
1. 从 quotes:batch 交叉匹配 sector:components 中的成分股，获取各股 pct
2. 按 pct 降序排列，找到该股的 1-based rank
3. pct_rank = rank / total  （0.01 = 行业第1，0.99 = 垫底）
4. 若找不到该股代码 → pct_rank = 0.5（中位）
```

#### Part 2 — 行业统计

```python
all_pcts = [匹配到的成分股 pct 列表]
median_pct = median(all_pcts)
pct_std = std(all_pcts)
```

#### Part 3 — 历史5日非涨停日估算

排除涨停日（pct ≥ 9.9%），取最近 5 个日常交易日。对每一天通过正态分布近似估分位：

```python
z_score = (daily_pct - median_pct) / pct_std

# 正态 CDF 近似查表：
if z < -3:   estimated_rank = 0.99    # 垫底
elif z < -1: estimated_rank = 0.84 + (z + 1) * 0.16
elif z < 0:  estimated_rank = 0.50 + z * 0.34
elif z < 1:  estimated_rank = 0.50 - z * 0.34
elif z < 3:  estimated_rank = 0.16 - (z - 1) * 0.16
else:        estimated_rank = 0.01    # 头部

estimated_rank = clamp(estimated_rank, 0.01, 0.99)
```

5 日分位取平均。

> 注：此处用当日 median_pct / pct_std 作为历史日分布的近似。假设板块成分股的相对分布短期内稳定。

#### Part 4 — 加权平均

```python
avg_rank = pct_rank × 0.6 + avg(estimated_ranks) × 0.4
```

#### Part 5 — 偏离度加分

```python
deviation = latest_pct - median_pct
deviation_bonus = clamp(deviation / pct_std × 10, 0, 20)
```

比行业中位数高出几个标准差 → 加分，上限 +20。

#### Part 6 — Lead-Lag 检测（新增）

对比个股 5 分 K 和板块 5 分 K 的时序关系：

```python
# 对每根 5 分 bar，检测"个股先拉、板块后跟"的模式
lead_lag_score = 0

for each_bar_window:
    # 个股在该窗口显著上涨（pct > threshold），板块滞后 N 根 bar 后跟随
    if stock_bar.pct > 0.5 and sector_later_bar.pct > 0.3:
        # 计算领先幅度和跟随紧密程度
        lead_lag_score += ...

# 归一化到 0-20
lead_lag_bonus = clamp(lead_lag_score, 0, 20)
```

具体阈值和算法在实现时细化，核心思想：
- 个股 5 分 K 涨幅 > 0.5%（启动信号），同时板块该 bar 涨幅 < 0.3%（尚未跟）
- 接下来 6 根 bar（30 分钟）内板块出现涨幅 > 0.3%，且之前个股涨幅明显跑赢板块
- 如果个股拉升了、板块死活不跟 → 减分

#### 最终得分

```python
leading_score = clamp((1 - avg_rank) × 100 + deviation_bonus + lead_lag_bonus, 0, 100)
```

### 4.4 边界情况

| 情况 | 处理 |
|------|------|
| 板块成分股仅 1 只（自己） | pct_rank = 0.5 |
| 成分股列表为空 | 50 分，fallback=True |
| quotes:batch 无该股数据 | pct_rank = 0.5 |
| 无板块 5 分 K（lead-lag 不可用） | lead_lag_bonus = 0 |

### 4.5 输出

```python
ScoreResult(
    dim="leadership",
    score=78.0,
    weight=0.25,
    details={
        "intraday_rank": 3,
        "total_components": 45,
        "intraday_percentile": 0.956,
        "avg_historical_rank_pct": 0.12,   # 5日平均排位
        "weighted_avg_rank": 0.07,          # 加权后排位
        "deviation_bonus": 5.0,
        "lead_lag_bonus": 8.0,
        "sector_median_pct": 1.2,
        "sector_std": 2.5,
    }
)
```

## 五、资金承接性 scorer（权重 25%）— `scorers/absorption.py`

### 5.1 核心问题
市场恐慌时，其他板块的资金是不是虹吸到了这个板块？

### 5.2 输入数据

| 缓存 key | 内容 | 用途 |
|----------|------|------|
| `kline:5min:sector:{s_code}` | 20 个板块的 5 分 K（10 涨 + 10 跌） | 跨板块资金流对比 |
| `sector:components:{s_code}` | 板块成分股列表 | 辅助 |

### 5.3 算法步骤

#### Step 1 — 滑动窗口检测虹吸事件

**参数**：窗口 6 根 bar（30 分钟），步长 1 bar，扫描全天。

**事件必须同时满足 4 个条件**：

```
条件① — 资金出逃：≥2 个其他板块 窗口内跌 > 1%
条件② — 资金涌入：目标板块 窗口内涨 > 0.3%
条件③ — 方向一致：≥4/6 根 bar 为阳线 (close > open)
条件④ — 不跳水：窗口最高点到收盘回撤 < 30% 窗口涨幅
```

**窗口涨跌幅**：

```python
for each_other_sector:
    window_return = close[t+5] / open[t] - 1
    if window_return < -0.01:
        mark_as_fleeing()

target_window_return = close[t+5] / open[t] - 1

up_bars = count(close > open for bar in window)
if up_bars < 4:
    continue  # 有反抽，不是持续流入

peak = max(close in window)
retrace = (peak - final_close) / (peak - window_open)
if retrace > 0.3:
    continue  # 冲高回落，假流入
```

#### Step 2 — 事件打分

每个事件三维评估：

```
event_score = intensity × 0.40 + breadth × 0.20 + sustain × 0.40
```

##### (a) 虹吸强度 Intensity（40%）

```python
flight_intensity = abs(出逃板块平均跌幅) × 出逃板块数量
intensity_score = min(target_rise / (flight_intensity + 0.001) × 100, 100)
```

- 同时考虑目标涨幅和出逃端的严重程度
- 出逃板块跌越多、目标板块涨越多 → 虹吸信号越强烈

##### (b) 虹吸广度 Breadth（20%）

```python
breadth_score = min(被抽血板块数量 / 10, 1.0) × 100
```

被抽血板块越多 → 资金从更广泛的地方集中过来 → 信号越强。

##### (c) 持续性 Sustain（40%）

```python
sustain_score = (1 - retrace) × 100
```

- 回撤 0% → 100 分（钱进来就不走了）
- 回撤 30% → 70 分（有部分流出）

#### Step 3 — 汇总

```python
best = max(event_scores)
multi_bonus = min((event_count - 1) × 5, 15)  # 多次虹吸，最多 +15
score = min(best + multi_bonus, 100)
```

**无事件时**：返回 **50 分**（中性），details 注明 `fallback_reason: "暂无显著的跨板块资金虹吸信号"`。

> 理由：找不到虹吸信号 ≠ 板块差，可能是市场平稳无恐慌。不做惩罚性降级。

### 5.4 边界情况

| 情况 | 处理 |
|------|------|
| 目标板块 5 分 K 数据不足 | 50 分（中性） |
| 无任何虹吸事件 | 50 分（中性） |
| 候选股来自领跌板块（不应出现） | 50 分 |
| 尾盘 bar 不够（窗口靠近收盘） | 回撤用剩余 bar 计算 |
| 部分板块 5 分 K 缺失 | 用已加载的板块做检测 |

### 5.5 输出

```python
ScoreResult(
    dim="absorption",
    score=82.0,
    weight=0.25,
    details={
        "event_count": 3,
        "best_event": {
            "time": "10:00-10:30",
            "target_pct": 2.1,
```

## 六、综合评分

在 `analyze.py` 中汇总：

```python
def analyze(code: str, cache: DataCache) -> dict:
    from dragon_quant.scorers.drive import score as drive_score
    from dragon_quant.scorers.anti_drop import score as anti_drop_score
    from dragon_quant.scorers.leadership import score as leadership_score
    from dragon_quant.scorers.absorption import score as absorption_score

    results = {
        "drive":      drive_score(code, cache),
        "anti_drop":  anti_drop_score(code, cache),
        "leadership": leadership_score(code, cache),
        "absorption": absorption_score(code, cache),
    }

    composite = sum(r.score * r.weight for r in results.values())

    return {
        "code": code,
        "composite_score": round(composite, 2),
        "dimensions": {k: {
            "score": v.score,
            "weight": v.weight,
            "details": v.details,
        } for k, v in results.items()},
    }
```

## 十、待解决：StockInfo 缺涨跌幅

**现状**：`eastmoney.py` 的 `get_sector_components` 返回的 `StockInfo` 对象只有 `(code, name, sector_code)`，尽管 API 响应中有 `f3`（涨跌幅）和 `f2`（最新价）字段，但未被填充。

**影响**：带动性和领涨性 scorer 需要板块成分股的涨跌幅，目前只能从 `quotes:batch` 按 code 做 O(n) 二次查表，增加复杂度且可能漏判（腾讯接口有 200 只上限）。

**解决方案**：
1. `StockInfo` 增加 `pct: float = 0.0` 字段
2. Provider 层 `get_sector_components` 填充 `f3` → `pct`
3. 带动性/领涨性 scorer 直接从 `sector:components` 取涨跌幅
